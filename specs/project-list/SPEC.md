# project-list — SPEC

> **状态**：已完成
> **模块**：project-list
> **最后更新**：2026-04-19

## 1. 概述

项目列表页是墨染 V2 的启动台（路由 `/` 或 `/projects`），用户在此创建新项目、进入已有项目、或与墨衡闲聊。核心交互入口是底部的聊天输入框——墨衡识别用户意图后自动路由。

技术栈：Next.js 页面组件 + Zustand UI store + SSE 事件订阅。

## 2. 功能需求

### 2.1 页面布局

- 顶部：品牌名"墨染 MoRan" + 右上角用户头像
- 中部：项目卡片区（按最后活跃时间倒序排列）
- 底部：聊天输入框（墨衡轻量对话入口）
- 无项目时中部显示空状态引导（示例文字可点击填入输入框）

### 2.2 项目卡片

每张卡片展示以下字段：

| 字段 | 说明 |
|------|------|
| 项目名 | 用户命名 |
| 题材标签 | 如"赛博朋克 × 修仙" |
| 阶段 | 筹备中 / 写作中 / 已完结 |
| 进度 | 已写章节数 / 大纲总章节数 |
| 字数 | 总字数统计 |
| 最后活跃 | 相对时间（如"2小时前"、"昨天"） |

**卡片交互**：
- 点击卡片 → 跳转该项目主工作页 `/projects/:id`，墨衡恢复上下文
- 右键/长按 → 项目操作菜单（重命名、置顶、导出、归档、删除）
- 卡片按最后活跃时间倒序排列

### 2.3 聊天输入框意图路由

项目列表页的输入框是轻量级墨衡对话入口，墨衡识别意图后自动路由：

| 用户输入示例 | 识别意图 | 动作 |
|-------------|---------|------|
| "我想写一本赛博朋克修仙小说" | 创建新项目 | 自动创建项目 → 跳转主工作页 → 开始引导 |
| "来一本末日题材的" | 创建新项目 | 同上，墨衡追问补充信息 |
| "继续写赛博修仙" | 进入已有项目 | 模糊匹配项目名 → 跳转主工作页 → 恢复上下文 |
| "上次那个末日武侠写到哪了？" | 查询项目状态 | 匹配项目 → 内联回复进度摘要 |
| "你好" / "最近怎么样" | 闲聊 | 内联回复，不跳转 |

**意图路由逻辑**：
1. 创建新项目意图 → 提取题材/类型信息 → 信息充足则自动创建跳转，不足则内联追问
2. 进入已有项目 → 模糊匹配 → 唯一匹配则跳转，多个则列出候选，无匹配则提示
3. 查询或闲聊 → 内联回复，不跳转

### 2.4 内联回复

- 列表页默认**不展示任何聊天消息**，保持启动台简洁
- 用户发送消息后，墨衡回复以气泡形式在输入框上方向上生长
- 最多保留最近 **3 轮对话**
- 跳转到项目主工作页后，列表页气泡自动清除
- 列表页对话**不带入**主工作页（主工作页有独立的项目级 session）

### 2.5 创建项目流程

项目创建通过对话自然完成，不使用独立的弹窗表单：

1. 用户给出题材 + 核心设定 → 直接创建，进入脑暴
2. 用户只给模糊方向 → 追问 1-2 轮补充关键信息
3. 用户说"随便写点什么" → 墨衡主动推荐题材方向

墨衡根据用户提供的信息密度决定追问深度。

### 2.6 项目操作菜单

右键点击项目卡片弹出上下文菜单：

| 操作 | 效果 |
|------|------|
| 📝 重命名 | 卡片标题变为内联编辑输入框 |
| 📌 置顶 | 卡片移到最前，显示置顶标记 |
| 📥 导出 | 打开导出对话框 |
| 📦 归档 | 确认对话框 → 移到归档区 |
| 🗑️ 删除 | 二次确认（输入项目名）→ 永久删除 |

### 2.7 空状态

无历史项目时的空状态展示：
- 居中文案："✨ 还没有项目 / 告诉墨衡你想写什么故事吧"
- 示例文字："我想写一本赛博朋克修仙小说"、"帮我续写上次的末日武侠"
- 示例文字**可点击**，点击后填充到底部输入框

### 2.8 用户菜单

右上角用户头像点击弹出下拉菜单：
- 个人设置（进入个人设置页面）
- 使用统计（进入使用统计页面）
- 退出登录（清除 session，跳转登录页）
