# info-panel — SPEC

> **状态**：已完成
> **模块**：info-panel
> **最后更新**：2026-04-19

## 1. 概述

信息面板是主工作页（`/projects/:id`）的右侧区域，通过 8 个 Tab 页展示各创作阶段的结构化产出。所有 Tab 内容**只读展示**，修改必须通过左侧聊天与墨衡对话完成（确保变更经过门禁校验）。面板数据通过 SSE 事件自动更新。

技术栈：Next.js 组件 + Zustand UI store + SSE 事件订阅 + 图表库（雷达图/趋势图）+ 关系可视化库（技术选型待定，不使用 Mermaid）。

## 2. 功能需求

### 2.1 Tab 栏设计

**8 个 Tab**：脑暴、设定、角色、大纲、章节、审校、分析、知识库

- Tab 栏固定面板顶部，不随内容滚动
- 当前激活 Tab 加下划线高亮
- Tab 名称 14px 字体，间距 24px

#### Tab 可见性

Tab 根据项目阶段动态出现（有数据才显示）：

| 阶段 | 可见 Tab |
|------|----------|
| 新项目（无数据） | 无 Tab，显示空状态引导 |
| 脑暴阶段 | [脑暴] |
| 设定阶段 | [脑暴] [设定] [角色] |
| 大纲阶段 | [脑暴] [设定] [角色] [大纲] |
| 写作阶段 | 全部 8 个 |

#### 徽标

- 🔴 红点：有新内容未查看（切到该 Tab 后消除）
- 数字角标：如 `[审校 ②]`（2 份未读报告）
- 🔴 实时标识：`[章节🔴]`（执笔正在写作）

#### 自动切换与 10 秒保护

子 Agent 产出新内容时面板自动切 Tab，但有 10 秒保护机制——用户 10 秒内有操作（点击、滚动、选中文字）则不切换，仅加红点。

```typescript
function handleAutoSwitch(targetTab: TabId) {
  const now = Date.now();
  const lastUserAction = uiStore.getState().lastUserActionTime;
  if (now - lastUserAction < 10_000) {
    tabStore.addBadge(targetTab, "dot"); // 不打断
  } else {
    tabStore.setActive(targetTab);       // 自动切换
  }
}
```

#### 面板空状态

项目刚创建、无 Agent 产出时显示引导："📝 开始你的创作之旅 / 在左侧与墨衡对话..."

### 2.2 [脑暴] Tab

**数据来源**：灵犀

按三阶段组织：**发散 → 聚焦 → 结晶**，每阶段可折叠。

- **发散区域**：方向列表，每项可加 ⭐ 星标（点击自动发送"我喜欢方向 X"到聊天）
- **聚焦区域**：入选方向、题材、核心冲突、目标读者
- **结晶方案**：卡片形式突出展示（类型、核心概念、卖点、体量、一句话梗概）

空状态："还没有脑暴记录。在左侧告诉墨衡你的创作灵感..."

**SSE 事件**：`brainstorm.diverge` → 追加方向；`brainstorm.converge` → 更新聚焦；`brainstorm.crystallize` → 渲染结晶卡片 + 触发自动切 Tab。

### 2.3 [设定] Tab

**数据来源**：匠心

采用**分类标签 + 卡片网格 + 子系统详情页**三层结构。

#### 总览视图（默认）

- 顶部搜索框：全文搜索所有子系统（debounce 300ms，关键词高亮）
- 动态分类标签（filter chips）：匠心根据题材自动生成，不硬编码。溢出时 `[+N]` 展开
- 卡片网格：每张卡片=一个子系统（图标+名称+关键词摘要+条目数+最后更新章节）
- 有新内容时卡片右上角 🔴 红点

面板宽度自适应：≥500px 双列 / 300-499px 单列 / <300px 紧凑列表。

**分类标签由题材决定**（示例）：
- 仙侠：力量体系、宗门势力、种族、地理、历史、神话传说……
- 都市：社会背景、势力关系、经济体系、科技水平……
- 科幻：科技树、星际地理、外星种族、政治体制……

**内置术语表**：「术语表」作为不可删除的内置分类始终存在。匠心设计世界观时自动提取关键术语（专有名词、力量等级、地名等），博闻在写作过程中持续维护。

#### 子系统详情页

点击卡片 → Tab 内容区切换为详情页：

- 面包屑导航：`← 返回设定总览 › 子系统名`
- 折叠分段：按主题分段，每段可折叠（`▼`/`▶`），默认全展开
- 段内卡片：内容在卡片容器内，层次清晰
- 条目计数：折叠标题后 `(N 条)`
- 关联引用：底部列出与其他子系统的交叉引用，点击跳转
- 更新来源：顶部标注最后更新章节和 Agent 名

空状态："世界观设定尚未创建。当脑暴方案确定后..."

**SSE 事件**：`world.created` / `world.subsystem_created` / `world.subsystem_updated` / `world.subsystem_deleted`。

### 2.4 [角色] Tab

**数据来源**：匠心

#### 双维度分类

- **叙事功能**（筛选）：主角 / 第二主角 / 对手 / 配角 / 次要
- **设计深度**（分组展示）：核心层 / 重要层 / 支撑层 / 点缀层

筛选器选中某叙事功能后仅显示该功能角色，但仍按层级分组。

#### 四层级展示

| 层级 | 卡片内容 | 心理模型 |
|------|---------|---------|
| **核心层** | 完整：性格画像+能力+弧光+心理模型+关系+当前状态 | ✅ 五维 GHOST/WOUND/LIE/WANT/NEED |
| **重要层** | 简要：性格+目标+关系+弧光概述 | 推荐（对手必填） |
| **支撑层** | 摘要：传记+核心特征+主要关系 | — |
| **点缀层** | 无独立卡片，折叠为索引列表（姓名+一句话） | — |

核心层/重要层默认展开，支撑层/点缀层默认折叠。

#### 核心层详情展开

点击 [查看详情] 原地展开：基础信息、性格画像、能力、**心理模型五维**（GHOST→WOUND→LIE→WANT↔NEED）、人物弧光、关系网络、当前状态。

**当前状态**为结构化区块，由载史每章归档时更新：
- 位置（当前所在地）
- 情绪状态（当前主要情绪）
- 新获知识/能力（本章新获得的信息或能力）
- LIE 进度（核心谎言的动摇/强化程度）
- 关系变化（本章发生的关系变动摘要）

#### 角色关系网络

展开后以**可交互关系图**渲染（仅核心层+重要层角色）：
- 节点可拖拽
- 悬停高亮关联边
- 点击节点跳转角色卡片
- 支持缩放平移
- 边标注关系类型

> **⚠️ 技术选型待定**：不使用 Mermaid。候选：D3.js force graph、React Flow、Cytoscape.js、G6。一套方案覆盖角色关系、势力结构、世界设定关联等所有关系图场景。

空状态："角色档案尚未创建。世界观设定完成后..."

**SSE 事件**：`character.created` / `character.updated` / `character.state_changed` / `character.tier_changed`。

### 2.5 [大纲] Tab

**数据来源**：匠心

#### 三视图切换

Tab 内顶部切换栏：**[大纲]** | [伏笔追踪] | [时间线]

##### 大纲视图（默认）

**弧段 → 章节**两级树形结构：
- 弧段可折叠（标题+章节范围）
- 章节节点点击原地展开 Brief 详情（剧情摘要、核心事件、伏笔/爆点、涉及角色、字数目标）

**状态标记**：

| 图标 | 状态 |
|------|------|
| ✅ | 已完成（审校通过并归档） |
| 📝 | 写作中 |
| 🔄 | 审校中 |
| ⏳ | 待写（有 Brief） |
| 📋 | 待规划（占位，无 Brief） |

##### 伏笔追踪视图

按伏笔状态分组展示：
- **🔴 未解决**：已埋未回收的伏笔（标注埋设章节、涉及角色、预计回收弧段）
- **✅ 已回收**：已解决的伏笔（标注埋设章节 → 回收章节）
- **⚠️ 超期**：超过预计回收弧段仍未解决的伏笔（高亮警告）

每条伏笔可点击展开详情（描述、关联角色、埋设位置引用）。

##### 时间线视图

按故事内时间轴展示事件流：
- 竖向时间轴，每个节点 = 一个故事事件
- 节点标注：时间点、事件描述、涉及角色、对应章节
- 支持按角色筛选（仅显示该角色参与的事件）
- 并行事件（同一时间不同地点）横向并列展示

空状态："故事大纲尚未创建。角色设定完成后..."

**SSE 事件**：`outline.arc_created` / `outline.brief_created` / `outline.brief_updated` / `outline.status_changed` / `outline.foreshadow_added` / `outline.foreshadow_resolved` / `outline.timeline_updated`。

### 2.6 [章节] Tab

**数据来源**：执笔

两种模式：**阅读模式**（浏览已完成章节）和**写作模式**（实时观看写作）。

#### 阅读模式

- 章节列表（可折叠侧栏）：点击切换阅读内容，选中高亮
- 元数据栏：字数、审校状态、写作风格
- 正文区域：只读展示

#### 写作模式（执笔写作时自动进入）

- Tab 标识：`[章节🔴]`
- 进度条：已写字数 / 目标字数 + 百分比
- 流式渲染：文字逐字出现（SSE `chapter.token` 驱动）
- 光标闪烁：文末 `█`
- 自动跟随：默认自动滚到最新，用户手动滚动后停止，底部出现 `[↓ 回到最新]`
- 完成后：移除 🔴，进度 100%，自动切回阅读模式

空状态："还没有章节内容。大纲完善后，告诉墨衡'开始写第一章'..."

**SSE 事件**：`chapter.start` / `chapter.token` / `chapter.complete` / `chapter.archived`。

### 2.7 [审校] Tab

**数据来源**：明镜

按章节组织，每次审校包含**四轮维度评分**。

#### 布局

- 章节选择下拉
- 综合评分卡片（总分、结论✅/⚠️/❌、四轮评分条）
- 四轮详情折叠区域（展开显示具体问题列表：位置、描述、建议、严重程度🔴/🟡）
- 历史记录：各轮次对比

#### 审校结论

| 结论 | 条件 | 后续 |
|------|------|------|
| ✅ 通过 | 总分 ≥ 80 且无 🔴 | 归档 → 下一章 |
| ⚠️ 修改后通过 | 总分 ≥ 70 有 🟡 | 执笔针对性修改 |
| ❌ 需重写 | 总分 < 70 或有 🔴 | 执笔重写 |

空状态："还没有审校报告。章节写作完成后..."

**SSE 事件**：`review.started` / `review.round_complete` / `review.complete`。

### 2.8 [分析] Tab

**数据来源**：析典

#### 双视图切换

Tab 内顶部切换栏：**[本项目分析]** | [参考作品]

##### 本项目分析视图（默认）

**九维雷达图 + 趋势折线图 + 详细评语**。

#### 九个维度

情节张力、角色塑造、对话质量、描写质量、原创性、主题呼应、伏笔管理、氛围营造、节奏控制。

#### 图表

- **雷达图**：九维分布，中心标注综合分，悬停 Tooltip 显示具体分数
- **趋势图**：X 轴章节序号，Y 轴 0-100，多折线（可通过图例切换显示/隐藏）
- **详细评语**：可折叠的析典完整文字分析

##### 参考作品视图

展示析典对外部作品的分析报告（用户通过聊天请求，如"帮我分析《诡秘之主》的伏笔技巧"）。

- 作品列表：卡片式展示（作品名 + 分析主题 + 分析日期）
- 点击展开：完整分析报告（Markdown 渲染）
- 底部提示：可操作经验已自动存入 [知识库] Tab

**数据流**：用户请求 → 墨衡 → 析典分析 → ① 报告存入 analysis 表（scope: external）→ 参考作品视图；② 可操作经验存入 knowledge_entries（category: analysis）→ 知识库 Tab。

空状态："还没有分析数据。章节归档后..."

**SSE 事件**：`analysis.complete` / `analysis.external_complete`。

### 2.9 [知识库] Tab

**数据来源**：博闻 + 析典（外部分析沉淀）

**定位**：纯创作技法层（Layer Ⅰ）——指导"怎么写"的经验知识。故事内容记忆（角色状态、世界设定、伏笔等）分散在各自对应的 Tab。

#### 五个分类

| 分类 | 说明 | 来源 |
|------|------|------|
| 写作技巧 | 通用创作技法（节奏控制、张力构建等） | 预置 + 用户补充 |
| 题材知识 | 题材专项技法（仙侠的修炼体系写法、悬疑的线索布局等） | 预置 + 匠心提取 |
| 风格专项 | 当前文风的细化指南（当前子写手的进阶技巧） | 预置 + 用户调整 |
| 经验教训 | 写作过程中与用户沟通产生的项目级教训 | 用户反馈 + 墨衡记录 |
| 析典沉淀 | 析典分析外部作品后提取的可操作经验 | 析典外部分析 |

#### 布局

- 搜索框：实时过滤（debounce 300ms）
- 分类筛选按钮：[全部] [写作技巧] [题材知识] [风格专项] [经验教训] [析典沉淀]，支持多选
- 范围筛选：[当前项目] / [全局]（切换查看项目级或全局知识）
- 条目卡片列表：图标+分类+标题+摘要+来源+范围标签，点击展开完整内容
- 分页：每页 20 条，[加载更多 ▼]

#### 知识条目展开

展示完整内容：定义、规则/技巧说明、来源（章节/外部作品/用户输入）、维护者、最后更新。

#### 提升为全局

项目级知识条目可通过 [↑ 提升为全局] 按钮复制到全局范围，使其对所有项目可用。

空状态："知识库暂无条目。随着创作推进，写作技巧和经验教训会在这里积累..."

**SSE 事件**：`knowledge.entry_added` / `knowledge.entry_updated` / `knowledge.promoted`。

### 2.10 面板 SSE 数据更新原则

- **数据更新与 Tab 切换解耦**：即使 Tab 未激活，后台数据也已更新
- **增量更新**：仅更新变化部分，不重载整个 Tab
- **离线缓存**：面板数据持久化到 IndexedDB
- **Tab 徽标**：有新更新但用户未查看时显示红点或数字
