# chat-ui — SPEC

> **状态**：已完成
> **模块**：chat-ui
> **最后更新**：2026-04-19

## 1. 概述

聊天窗口是主工作页（`/projects/:id`）的左侧面板，用户与墨衡的唯一对话入口。包含消息流、输入区、Agent 状态条、Question Panel、底部快捷栏和顶部导航栏。章节正文不在聊天窗口展示，而在右侧信息面板 [章节] Tab 流式渲染。

技术栈：Next.js 页面组件 + Zustand UI store + SSE 事件订阅。

## 2. 功能需求

### 2.1 主工作页整体布局

左右分栏，中间可拖拽分隔条：

| 屏幕宽度 | 会话窗口（默认） | 信息面板（默认） | 交互模式 |
|----------|------------------|------------------|----------|
| ≥ 1440px | 40% | 60% | 并排显示，可拖拽 |
| 1024-1439px | 45% | 55% | 并排显示，可拖拽 |
| 768-1023px | 50% | 50% | 并排显示，可拖拽，可折叠面板 |
| < 768px | 100% | 100% | Tab 切换（聊天/面板），无分隔条 |

**拖拽行为**：
- 分隔条宽度 4px，hover 时变为 8px 并高亮（`cursor: col-resize`）
- 拖拽范围：左侧最小 25%，右侧最小 30%
- 比例持久化到 `localStorage`，下次进入同一项目时恢复
- 拖拽过程中双面板实时 resize（不遮挡、不延迟）
- 双击分隔条 → 恢复默认比例

### 2.2 消息类型

| 类型 | 来源 | 展示方式 |
|------|------|----------|
| 用户消息 | 用户输入 | 右对齐气泡，浅色背景 |
| 墨衡消息 | 墨衡回复 | 左对齐气泡，白色背景，支持流式打字机效果 |
| 系统通知 | 子 Agent 完成/状态变更 | 居中小字，灰色（如"灵犀完成了脑暴"） |
| 进度指示 | 子 Agent 工作中 | 左对齐，带动画的进度条/状态文字 |
| 决策提问 | 墨衡需要用户决策 | 触发 Question Panel 替换输入框 |

墨衡消息支持富文本：Markdown 渲染、评分表、内联按钮（如 `[查看详细报告]`、`[进入下一章]`）。

### 2.3 输入区

| 组件 | 功能 |
|------|------|
| 输入框 | 多行文本，Shift+Enter 换行 |
| 发送按钮 | Enter 或点击发送 |
| 📎 附件按钮 | 上传参考资料（.txt, .md, .docx, .pdf，单文件 10MB） |
| `/` 快捷命令 | 键入 `/` 触发命令面板 |

### 2.4 快捷命令

| 命令 | 功能 |
|------|------|
| `/write [N]` | 开始写第 N 章 |
| `/review [N]` | 审校第 N 章 |
| `/status` | 查看项目总体进度 |
| `/export` | 导出已完成章节 |
| `/brainstorm` | 开始新一轮脑暴 |
| `/analyze [N]` | 析典分析第 N 章/弧段 |
| `/lesson` | 查看/添加写作教训 |
| `/style` | 查看/调整文风 |
| `/rollback [N]` | 回滚到第 N 章的某个版本 |

### 2.5 流式输出

会话窗口只有一种流式输出：**墨衡对话流式**——打字机效果逐字渲染。执笔写作的流式输出在右侧 [章节] Tab，Agent 状态在状态条，均不进入聊天流。

### 2.6 决策交互面板（Question Panel）

当墨衡需要用户选择决策时，底部输入区**临时替换**为问题选择面板：

- 卡片式布局，每个选项一行，点击即选择
- 最后一个选项始终为"💬 自由输入"——点击后恢复普通输入框
- 选中项作为用户消息发送
- 发送后输入区自动恢复默认模式
- 支持键盘快捷键：数字键 1-9 直接选择
- **触发条件**：墨衡回复携带 `interaction_mode: "question"` metadata + `options` 数组

**常见决策场景**：弧段边界、审校未通过、脑暴方案选择、Lesson 确认、写手选择。

### 2.7 Agent 工作状态条

位于输入框**正上方**，展示当前正在工作的子 Agent：

**状态指示灯**：

| 颜色 | 状态 | 说明 |
|------|------|------|
| 🟢 绿色 | active | 正在执行，有实时输出 |
| 🟡 黄色 | queued | 等待前序 Agent 完成 |
| 🔵 蓝色 | background | 后台处理（归档、分析） |
| ⚪ 灰色 | just_finished | 短暂显示 3 秒后淡出 |

**文案模板**（按 Agent）：
- 执笔·{子名}：`写作第 N 章 · {字数}字` / `✅ 完成第 N 章（{总字数}字）`
- 明镜：`审校第 N 章 · 第 M 轮` / `✅ 审校完成 {评分}分`
- 灵犀：`脑暴方案生成中...` / `✅ 脑暴完成（{N}个方案）`
- 匠心：`设计{目标}...` / `✅ {目标}设计完成`
- 载史/博闻/析典：类似格式

**布局规则**：
- 无 Agent 工作时完全隐藏（高度归零）
- 有工作时平滑滑入（200ms ease-out）
- 最多显示 2 行，超出折叠为"+N 个 Agent 工作中"，hover 展开完整列表
- 点击任一 Agent 行 → 滑出 Agent 会话抽屉

**SSE 驱动**：
- `subtask_start` → 新增状态行
- `subtask_progress` → 原地刷新描述文案（数字平滑过渡）
- `subtask_end` → 灰色 + 完成文案，3 秒后淡出
- `writing_progress` → 高频推送（~500ms），实时刷新字数
- 断线重连 → `GET /api/projects/:id/agent-status` 拉取当前全部状态

**数据模型**：

```typescript
interface AgentStatus {
  agentId: string;           // "zhibi", "mingjing", etc.
  displayName: string;       // "执笔·剑心", "明镜", etc.
  state: 'active' | 'queued' | 'background' | 'just_finished';
  description: string;       // "写作第 38 章 · 1,847字"
  startedAt: number;
  targetTab?: string;        // 点击跳转的 Tab
}
```

### 2.8 Agent 会话抽屉

点击状态条中的 Agent 行，从右侧滑出抽屉：

- 宽度固定 320px，覆盖在信息面板之上（不挤压布局）
- 顶部：Agent 名称 + 任务描述 + ✕ 关闭按钮
- 内容：该 Agent 的会话消息流，SSE 驱动实时渲染
- Agent 完成后抽屉不自动关闭
- 点击外部或按 Esc → 关闭
- 同时只能打开一个抽屉，点击另一个 Agent → 切换内容

**可查看会话的 Agent**：明镜（审校推理）、书虫（阅读体验）、灵犀（脑暴过程）、匠心（设计推理）、析典（分析细节）、博闻（校验矛盾）、执笔（指令交互）。

### 2.9 顶部导航栏

```
← 返回列表 │ 📖 项目名 │ 阶段 │ 进度 │ 字数 │ Token消耗 │ ⚙ 设置
```

| 元素 | 功能 |
|------|------|
| ← 返回 | 回到项目列表页 |
| 📖 项目名 | 显示当前项目名，点击可内联编辑（Enter 保存，Esc 取消） |
| 阶段 | 筹备中 / 写作中 / 已完结 |
| 进度 | 已完成章节 / 大纲总章节 |
| 字数 | 累计字数 |
| Token 消耗 | 本项目累计花费，点击弹出消耗摘要，含 [查看详细报表] 链接 |
| ⚙ 设置 | 项目设置抽屉（400px 右侧滑出） |

### 2.10 底部快捷操作栏

5 个按钮，本质是向墨衡发送预定义消息：

| 按钮 | 等效消息 |
|------|----------|
| 📝 继续写作 | "继续写下一章" |
| 🔍 送审校 | "审校最新章节" |
| 📊 查看进度 | "给我项目进度报告" |
| 📥 导出 | 打开导出对话框 |
| ⏸ 暂停 | "暂停当前工作" |

暂停操作：不弹窗，在聊天中显示墨衡确认 + Agent 状态条更新为 `⏸ 已暂停`。恢复：用户说"继续"。

### 2.11 二级视图

#### 项目设置抽屉

右侧滑入 400px，覆盖信息面板，包含：
- 基本信息（项目名编辑、题材、创建时间）
- 写作风格（当前风格、切换选择器）
- 模型配置（写作/审校/分析模型，可覆盖）
- 成本预算（已消耗、预算上限、达限行为）
- 写作参数（目标字数、审校阈值、每日上限）
- 危险操作（归档、删除）

#### 文件上传弹窗

📎 按钮触发，支持 .txt/.md/.docx/.pdf，单文件 10MB，拖拽上传。

#### 导出对话框

格式选择（TXT/Markdown/DOCX/EPUB）+ 范围选择 + 包含内容勾选。

#### Token 详细报表

600px modal，按 Agent 分布 + 按章节分布 + 费用趋势折线图。

#### 项目名内联编辑

点击导航栏项目名 → 原地变为输入框 + ✓ ✕ 按钮。
